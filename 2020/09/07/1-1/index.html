零基础三步通过hexo和GitHube打造个人博客!

总的来说分为三步：

1 注册Github，安装Git、node

2 配置Git和node

3 配置hexo

详细阐述：

1.注册Github安装Git、node

  • 1.1注册Github

官网：https://github.com

按指令注册即可，如果卡在验证码环节可考虑换chrome浏览器

  • 1.2安装Git

    官网 https://git-scm.com

下载安装即可

  • 1.3 安装node

    官网 https://nodejs.org

下载安装即可

git和node下载如嫌太慢，可去XX软件园之类国内资源站下载。

2.配置Git

  • 2.1检查本机是否已经存在SSH keys

在Git内输入：

cd ~/.ssh

检测本机用户home目录下是否存在.ssh，不存在，转2.2，存在，转2.3

  • 2.2创建新的SSH keys(密钥)

Git内输入

ssh -keygen -t rsa -C “GitHub邮箱”

然后回车，这就开始将密钥邮箱存入默认地址默认文件

然后一路回车，相当于不设密码

最后会出现以“db 邮箱”结尾的一行代码，就代表成功了

  • 2.3在GitHub账户中添加自己的公钥

Git中运行以下命令：

clip <~/.ssh/id_rsa.pub>

这是将公钥加入系统剪贴板上

然后在GitHub的Account Settings界面下选 SSH keys，点 Ad SSH keys按钮，将公钥粘贴至文本框中，再点绿色的[Add key]按钮，宣告胜利！

  • 2.4测试是否可以通过SSH连接GitHub

Git内输入以下命令：

ssh -T git@github.com

接下来会弹一下结尾是yes/no的警告信息，直接输yes.

如果看到以下信息，就证明胜利了！

“Hi username! You’ve successfully authenticated, but GitHub does not provide shell access.”

注意，最后半句虽然有but、does not这种否定词，但那是无所谓的，我第一次连到这看没有提供什么shell access还以为失败了呢，哈哈^_^。

  • 2.5设置Git的用户名和用户邮箱

设置用户名：

Git内输入：

git config – globa user.name “你在github里的用户名”

设置用户邮箱：

Git内输入：

git config – globa user.email “你在github里的注册邮箱”

配置好这两样后，GitHub主页上就会显示由谁提交了该文件。

检查是否设置成功：

Git输入：

git config –list

如果成功就会列出现在的用户名和邮箱。

  • 2.6 检测node是否安装成功

Git输入：

node -v

若成功则返回版本号

npm -v

若成功则返回版本号

  • 2.7 安装淘宝的npm镜像源(cnpm)：

Git输入：

npm install -g cnpm –registry=https://registry.npm.taobao.org

  • 2.8 检测cnpm是否成功安装

等稀里哗拉一大段代码结束后Git输入如下代码检测cnpm是否成功安装：

cnpm

返回cnpm相关信息

cnpm -v

返回cnpm版本号

如果失败就重新安一下就好。

3 安装设置 hexo，发布第一篇blog!

  • 3.1安装hexo

Git下，输入

cnpm -install -g -hexo -cli

  • 3.2检测hexo安装是否成功

hexo -v

返回hexo版本号

【注意！如果win10下管理员打开cmd然后hexo -v出现“此系统禁止运行脚本”等报错字样，要以管理员的身份启动powershell，并实行set-ExecutionPolicy RemoteSigned命令，然后再跑hexo -v就不会报错了】

  • 3.3 建立blog文件夹

创建一个空文件夹预备存放blog相关文件，爱创建在哪创建在哪，创建后右键“GitBash here”即可，这样就是在这个目录下开始输入命令

否则就要在Git内通过 cd命令，转移到blog文件夹下操作

例如：
cd: c:\users\blog

“cd”命令就是切到cd后面那个目录里

  • 3.4检测是否进入blog文件夹

在blog文件夹目录下，输入

pwd

这意思是给出当前文件夹的目录，如果弹出目录就是blog文件夹的目录，就对了

  • 3.5初始化hexo

在blog目录下，输入

hexo init

然后经过一段时间的初始化，会copy一堆文件放在blog目录下，输入：

ls -l

这是查看初始化copy过来了哪些文件

  • 3.6启动hexo

输入

hexo s

默认会将blog放在

http://localhost:4000

也就是本地的4000端口上，打开浏览器输入以上地址，就能看到自己的blog网页了。

但是到这一步，还只是将blog部署到了本地，其他人无法通过互联网看到你的blog，所以接下来我们要将blog放在github上发布出去，让全网都能看到。

  • 3.7将blog部署到GitHub上，宣告最终胜利！

首先在blog目录下安一个hexo部署插件，Git输入:

cnpm install –save hexo-deployer-git

接下来找到blog文件夹中的 config.yml文件，用编译器（比如vscode, sublimetxt，甚至记事本等等）打开

直接拉到最后

会发现有个deploy目录

这时，输入几行字，使deploy目录最终长成这样

deploy:

  type: git

  repo: 你github的仓库（repositories）地址

  branch: master

然后保存。

再进入 Git，输入：

hexo d

这就是将blog部署到GitHub中

等一大段代码跳完后，胜利了！

  • 3.8输入自己的GitHub仓库地址，检测是否成功部署blog

去GitHub，点击Your repositories，左上角会是用户名/仓库地址，复制那段仓库地址到浏览器中打开，就是我们刚刚部署的blog！

注意，如果出现PC端连不上blog但手机端能连上的情况，可以重启家里的路由器，有可能就治好了。
